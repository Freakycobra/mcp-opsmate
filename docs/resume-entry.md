### mcp-opsmate — Infrastructure Automation MCP Terminal
**Personal Project** | July 2025 | Python 3.13, FastAPI, React 18, TypeScript, MCP, PostgreSQL, Redis, Docker

**What it is:** A production-grade AI infrastructure automation terminal that converts natural language commands from platform engineers and SREs into multi-step, DAG-structured execution plans, dynamically discovering and chaining tool calls across 7 MCP servers (Tavily, GitHub, Slack, Jira, AWS, PostgreSQL, Calculator) with deterministic mock mode for zero-config onboarding and seamless LIVE mode graduation for production use. Built as both a Rich TUI CLI and a React Web UI with real-time DAG visualization.

**Architecture:** Protocol-oriented architecture built on Anthropic's Model Context Protocol (MCP) — the orchestrator engine has zero direct dependencies on external APIs, communicating exclusively through the MCP protocol. Layered architecture with clear separation between the orchestration engine (domain logic), transport adapters (CLI, Web UI, API), and external tool integrations (MCP servers). PostgreSQL-backed state machine with 6 execution states enabling pause/resume of long-running workflows.

**Key technical contributions:**
- Built a multi-step execution orchestrator that converts natural language into DAG-structured operational plans, achieving <2s end-to-end latency for single-tool commands in MOCK mode through async DAG execution with `asyncio.TaskGroup` and topological sorting
- Implemented 7 MCP servers (Tavily, GitHub, Slack, Jira, AWS, PostgreSQL, Calculator) with deterministic mock mode using seeded Faker, enabling zero-config onboarding — same seed produces identical outputs for reproducible demos and testing, with schema parity validated between mock and live responses
- Designed a 4-tier error recovery system (retry with exponential backoff → circuit breaker → degraded execution → human escalation) using Redis-backed circuit breakers with OPEN/HALF_OPEN/CLOSED state machine, achieving fail-fast behavior after 5 consecutive failures with 30s recovery timeout
- Built a hybrid intent classifier combining regex entity extraction (service names, time ranges, thresholds, resource identifiers via fuzzy matching with Levenshtein distance <= 2) with GPT-4o semantic classification (structured JSON output via function calling, temperature 0.2), achieving >90% accuracy on infrastructure command parsing with <1s classification latency
- Implemented execution state persistence in PostgreSQL with SQLAlchemy 2.0 async ORM, writing diff-based updates after every step completion (<50ms write latency), with graceful SIGTERM handling that completes in-flight steps, persists state, and enables resumption on restart
- Created a Rich-based CLI with streaming SSE output (plan panels, syntax-highlighted JSON results, colored status tables, progress spinners) and an interactive React 18 Web UI with real-time DAG visualization using ReactFlow, chat interface with markdown rendering, execution history with filtering, and admin dashboard with mode switching
- Containerized the full stack with Docker Compose supporting MOCK/LIVE/Mixed execution modes with per-server granularity; implemented hierarchical configuration (defaults → config.yaml → env vars → CLI flags) using Pydantic-Settings with `OPS_MATE_*` prefix
- Designed Pydantic v2 models for the entire API contract with auto-generated OpenAPI docs, two-tier authentication (API key for standard endpoints, admin bearer token for `/admin/*`), secret redaction in all logs and audit trails, and Prometheus-compatible `/metrics` endpoint
- Wrote comprehensive async test suite (pytest-asyncio) with 30+ test cases covering API routes, intent classification, DAG execution, error recovery, state transitions, and mock factory validation; all external APIs fully mocked, mock MCP hub with 3 servers for isolated testing

**Impact:** Demonstrates deep expertise in async Python architecture, protocol-oriented design, production-grade error handling, LLM integration for practical automation, and the full lifecycle of building and shipping developer tools — from requirements specification and architecture design to implementation, testing, containerization, and CI/CD. Directly applicable to platform engineering, SRE tooling, and AI-powered automation initiatives. The project serves as a reference implementation for MCP-based multi-agent orchestration with enterprise-grade reliability patterns.
